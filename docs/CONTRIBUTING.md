# Contributing to GlucoTray

Thanks for your interest in contributing. Please read this before opening a PR.

## Branching

- Base your work on `dev`, not `main`
- Branch naming: `feature/name`, `fix/name`, `docs/name`

## Commits

Use conventional commits:

- `feat:` new feature
- `fix:` bug fix
- `docs:` documentation only
- `chore:` tooling, dependencies

## Pull Requests

- One feature or fix per PR
- Describe what changed and why
- PRs go into `dev` — never directly into `main`

## Code Style

- Prettier for formatting (config in repo)
- No comments in code
- TypeScript strict mode

## Reporting Issues

Open a GitHub Issue. Include OS, GlucoTray version and steps to reproduce.
